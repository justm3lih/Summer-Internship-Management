Why this Project? (Problem Statement)
Manual Document Tracking

Relying on paper-based forms (application letters, printed logbooks) risks damage (wet/torn) and loss.

Communication Breakdown

Inefficient coordination channels (emails, chats) between students, academic advisors, and company supervisors.

Academic Assessment Burden

Reviewing hundreds of physical logbooks and training reports manually takes weeks of coordinators' time.

Information Pollution

Students struggle to find correct instructions, resulting in repeated inquiries to coordinators.

The Solution: End-to-End Digitalization
100% Digital Workflow

Migrates all processes (from initial placement request to final report grading) into a single, unified database system.

Role-Based Access Control

Features dedicated portals with secure views and workflows for Students, Supervisors, and Coordinators.

AI-Powered Advisor Support

Reduces coordinator load by delegating standard rule enquiries to a custom-trained AI assistant.

Unified Database Flow

Keeps all student records (acceptance documents, supervisor ratings, daily logs) digitally signed and linked in the system.

System Architecture & Tech Stack
Frontend Application

Built with Next.js 14, TypeScript, and Tailwind CSS. Modern UI elements powered by Shadcn UI.

Backend REST API

Powered by ASP.NET Core Web API (.NET 8 / C#) utilizing Entity Framework Core ORM.

Database & AI

PostgreSQL database storage. Google Gemini API (gemini-2.5-flash-lite) for AI RAG chatbot capabilities.

🛠️ Biz Nasıl Yaptık? (Clean Decoupled APIs)

We designed a decoupled architecture where the frontend interacts strictly via secure REST JSON endpoints. DB schemas and relationships are defined via C# Code-First Migrations.

W

Workplace Supervisor

Approves student logbook entries on a daily basis. Fills out final program outcome (PO) evaluation scoring sheets.

Verifies placement details, checks daily logbook signatures, schedules jüri evaluations, and enters final grades.

Student Portal & Key Workflows
Digital Logbook & Forms

Interactive daily calendar interface. Generates official PDF application letters and secures SGK insurance uploads.

Word Count Checker (Min: 20, Max: 100)
Live Progress Tracker

Visual progress tracker showing the current status of the internship (Pending, Approved, Ongoing, Completed).

🛠️ Biz Nasıl Yaptık? (DOCX & State Lock)

We parsed CIU's official DOCX XML tags in C# using OpenXML, mapping database logs directly. Logbook input is validated server-side and locked via state machine once approved.

Supervisor Portal & Evaluation
Daily Sign-off & Monitoring

Supervisor reads daily submissions, approves or rejects logs, and monitors intern progress dynamically.

PO Evaluation Sheets

Standardized evaluation covering 11 academic Program Outcomes (POs) with radial scoring feeding to backend computations.

🛠️ Biz Nasıl Yaptık? (PO Math & Supervisor Security)

Supervisors are restricted via database relations. We designed a backend engine that maps 11 PO Likert scores to the database and averages results dynamically, preventing score tampering.

Coordinator Portal & Admin Settings
Placement & SGK Verification

Reviews student applications, logs SGK insurance documents digitally, and manages knowledge base resources for students.

Admin Panel: Custom Staff Privileges

Fark Yaratan Özellik: AI & RAG Integration
Retrieval-Augmented Generation (RAG)

Dynamically queries the database for staj guidelines and personal logs count, passing them as a context block to Gemini to block hallucinations.

Database-Linked Personal Assistant

Reads user-specific placement data in real-time, allowing students to ask, "How many days left?" or "Is my SGK approved?".

🛠️ Biz Nasıl Yaptık? (Custom RAG Pipeline)

We wrote custom C# service wrappers around the Gemini SDK. Before calling the LLM, we execute C# queries to pull the student's active state, injecting it as structured background prompt context.

Security Infrastructure & Database
Cookie & Token Security (JWT)

JWT tokens are stored in HttpOnly, SameSite, Secure Cookies to protect user sessions from XSS/CSRF extraction.

Protected Document Streams

PDF/Word files are stored outside the public web root. Custom ASP.NET controller streams serve them only after validating role authorization.

🛠️ Biz Nasıl Yaptık? (EF Core SQL Guard & JWT Claims)

We used EF Core parameterized queries to block SQL Injection. All controller endpoints validate owner IDs extracted directly from the signed JWT payload claims, preventing IDOR.
